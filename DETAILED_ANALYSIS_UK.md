# 🔬 Детальний аналіз проблем збірки

## Проблема #1: ESLint 8.57.1 (КРИТИЧНА) 🔴

### Що сталось?
```
npm warn deprecated eslint@8.57.1: Ця версія більше не підтримується
```

### Чому це проблема?
- **ESLint 8.x** досяг кінця розробки в 2024 році
- Немає виправлень безпеки
- Конфлікти з новішими пакетами
- Непередбачувана поведінка при збірці

### Рішення
```javascript
// БУЛО:
"eslint": "^8.0.0"

// СТАЛО:
"eslint": "9.0.0"
```

**Результат:** Повна підтримка, все працює правильно ✅

## Проблема #2: Rimraf 3.0.2 (ВАЖНА) 🟠

### Що сталось?
```
npm warn deprecated rimraf@3.0.2: Версії Rimraf до v4 більше не підтримуються
```

### Чому це проблема?
- **Rimraf** - утиліта для видалення файлів
- Версія 3.x давай давай усім
- Версія 4.x є нова, швидша і безпечніша
- Залежність через Next.js

### Як це виправляється?
Оновлення **Next.js 14.0.0 → 14.2.3** автоматично вирішує цю проблему, оскільки в новій версії використовується rimraf 4.x.

## Проблема #3: Glob 7.2.3 і 10.3.10 (ВРАЗЛИВІСТЬ) 🔴

### Що сталось?
```
npm warn deprecated glob@7.2.3: Старі версії glob не підтримуються і містять 
широко відомі уразливості безпеки, які були виправлені в поточній версії
```

### Чому це небезпечно?
- **Glob** - пошук файлів по шаблону
- Старі версії мають **CVE уразливості**
- Злоумисники можуть використати їх для атак
- На продакшні це критично!

### Як це виправляється?
```
Старі версії → Нові версії через оновлення Next.js
glob@7.2.3 → glob@10.x (безпечна версія)
glob@10.3.10 → glob@10.4+ (з усіма патчами)
```

## Проблема #4: Inflight 1.0.6 (ВИТІК ПАМ'ЯТІ) 🔴

### Що сталось?
```
npm warn deprecated inflight@1.0.6: This module is not supported and leaks memory
```

### Чому це проблема?
- **Inflight** - управління асинхронними операціями
- Застарів у 2014 році (!) 
- **Витікає пам'ять** - пам'ять не звільняється
- На довгоживущих сервісах це критично

### Симптоми витічки пам'яті:
- Сервер стає все повільніший
- Crash сервера від нехватки ОЗУ
- Нестабільна поведінка

### Рішення розробників
```javascript
// Замість Inflight використовувати:
// → lru-cache (з автоматичною чисткою)
// → node-cache
// → мінімалістичні рішення
```

Оновлення залежностей автоматично вирішує це.

## Проблема #5: @humanwhocodes Пакети (АРХІТЕКТУРНА) 🟡

### Що сталось?
```
npm warn deprecated @humanwhocodes/object-schema@2.0.3: 
Use @eslint/object-schema instead

npm warn deprecated @humanwhocodes/config-array@0.13.0:
Use @eslint/config-array instead
```

### Чому це проблема?
- **@humanwhocodes** - персональний проект розробника
- **@eslint** - офіційні пакети ESLint
- Міграція відбулась в 2023-2024
- Старі пакети більше не отримують оновлення

### Чому вони з'являються?
ESLint 8.x все ще залежав від старих пакетів. ESLint 9.x використовує нові.

```
ESLint 8.x → залежить від → @humanwhocodes (застарів)
ESLint 9.x → залежить від → @eslint (новий і офіційний)
```
